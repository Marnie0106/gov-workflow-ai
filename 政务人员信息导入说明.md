# 政务人员信息导入说明

## 可导入字段说明

| 字段名 | 数据库列名 | 是否必填 | 说明 |
|--------|-----------|---------|------|
| 工号 | `employee_id` | ✅ 必填 | 唯一标识，用于登录，如 D001、A001、L001 |
| 用户名 | `username` | ✅ 必填 | 系统唯一，可与工号相同或不同 |
| 姓名 | `display_name` | ✅ 必填 | 右上角显示的真实姓名 |
| 角色 | `role_key` | ✅ 必填 | 见下方角色说明 |
| 部门 | `department` | 建议填写 | 显示在姓名下方（如"城管执法队一分队"） |
| 密码 | `password` | 可选 | 当前演示未使用密码验证，默认 123456 |

## 角色代码说明

| 角色代码 | 角色名称 | 进入页面 |
|---------|---------|---------|
| `dispatcher` | 工单处置人员 | /dispatcher |
| `process_admin` | 业务流程管理员 | /process-admin |
| `leader` | 领导 | /leader |

⚠️ 注意：`citizen`（市民）不在政务人员表中，市民通过手机号注册，存储在单独的 `citizens` 表。

## 方式三：用 CSV 批量导入脚本

准备一个 CSV 文件，格式如下（第一行为表头）：

```csv
username,employee_id,role_key,display_name,department
dispatcher3,D003,dispatcher,陈伟,城管执法队三分队
admin2,A002,process_admin,刘芳,政务协调科
leader2,L002,leader,周建国,局办公室
```

然后在 `backend` 目录运行以下 Node 脚本（如命名为 `import-staff.js`）：

```js
const sqlite3 = require('sqlite3').verbose();
const fs = require('fs');
const path = require('path');

const db = new sqlite3.Database(path.join(__dirname, 'workflow.db'));
const csv = fs.readFileSync('staff.csv', 'utf-8').trim().split('\n');
const headers = csv[0].split(',');

db.serialize(() => {
  const stmt = db.prepare(`
    INSERT OR REPLACE INTO users (username, employee_id, role_key, display_name, department, created_at)
    VALUES (?, ?, ?, ?, ?, datetime('now'))
  `);
  for (let i = 1; i < csv.length; i++) {
    const row = csv[i].split(',');
    const obj = {};
    headers.forEach((h, idx) => obj[h.trim()] = row[idx]?.trim());
    stmt.run(obj.username, obj.employee_id, obj.role_key, obj.display_name, obj.department);
    console.log(`导入：${obj.display_name}（${obj.employee_id}）`);
  }
  stmt.finalize();
  console.log('导入完成');
});
```

运行：
```bash
cd E:\gov-workflow-ai\backend
node import-staff.js
```

## 登录方式

政务人员登录时：
1. 打开系统，点击 **政务人员**
2. 选择身份（处置员 / 管理员 / 领导）
3. 输入工号（如 `D001`）
4. 点击登录 → 系统自动跳转到对应工作台
5. 右上角显示真实姓名和所在部门

> **注意**：当前演示版本按工号查询，不校验密码。如需密码验证，需在 `server.js` 的登录接口中添加 `password` 字段校验。
